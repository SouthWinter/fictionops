# Audit Reference

## Common Gates

- `review-gate`: chapter-level readiness.
- `book-gate`: book-level closure and open items.
- `audit-continuity`: prior facts, callbacks, and state transfer.
- `audit-info`: information release and knowledge boundaries.
- `audit-characters`: character arcs, intelligence modes, and voice boundaries.
- `audit-style`: prose pattern risks and over-regularity.
- `doctor`: broad project health.

## Review Priorities

1. Blocking canon or continuity errors.
2. Character knowledge leaks.
3. Scene logic that relies on author explanation instead of visible pressure.
4. Repeated prose patterns that make chapters feel mechanically uniform.
5. Release and packaging issues.

## Output Discipline

Prefer concrete findings with file paths and next actions. Avoid rewriting large sections unless the user asks for prose-level revision.
